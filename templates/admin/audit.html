🔐 How does the tamper-evident audit chain work?

Every log entry is linked to the previous one using HMAC-SHA3-256. The chain hash of each entry is computed as:

chain_hash = HMAC-SHA3-256(key=SECRET_KEY, msg=prev_hash + JSON(payload))

If any entry is modified, deleted, or reordered, the chain hash will no longer match — instantly revealing tampering. Admin usernames and IP addresses are stored as one-way SHA-256 hashes — they cannot be reversed.
